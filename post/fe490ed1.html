创建JpaRepository

JpaRepository是SpringDataJpa给我们提供的数据库访问层的工具类, 里面提供了一些基本的CRUD操作, 这样的话一些简单的增删改查就不需要自己动手了. 下图为JpaRepository给我们提供的操作数据库的方法.

//JpaRepository泛型1为实体, 2为主键的类型
public interface ApplicationUserRepository extends JpaRepository<ApplicationUser, Integer> {
}

多对多关联(基本)

实现基本的多对多关联, 不使用复合主键

  1. 配置文件
    在配置文件中加上: spring.jpa.hibernate.ddl-auto=update

如果没有这张表启动项目后JPA会自动创建

  1. 实体类
//ApplicationUser实体类, 注意要重写toString否则会递归导致栈溢出
@Data
@Accessors(chain = true)
@NoArgsConstructor
@AllArgsConstructor
@Entity
@TypeDef(name = "luckyNumbers", typeClass = JsonStringType.class)
public class ApplicationUser {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Integer id;

    private String name;
    private Integer age;
    private String email;

    @Type(type = "luckyNumbers")
    private int[] luckyNumbers;

    //使用多对对注解关联
    @ManyToMany(fetch = FetchType.EAGER)
    /* joinTable指定关系表的表名: 此处关系表只有两个字段一个是user表的主键, 
        另一个是address表的主键, joinColumns为user表联查时关联的主键,
        inverseJoinColumns则为反向, 也就是address表中的主键
     */
    @JoinTable(
            name = "user_address",
            joinColumns = @JoinColumn(name = "user_id"),
            inverseJoinColumns = @JoinColumn(name = "address_id"))
    //在Address中@ManyToMany注解的mappedBy需要和该属性名一致
    List<Address> addressList;

    @Override
    public String toString() {
        return "ApplicationUser{" +
                "id=" + id +
                ", name='" + name + '\'' +
                ", age=" + age +
                ", email='" + email + '\'' +
                ", luckyNumbers=" + Arrays.toString(luckyNumbers) +
                '}';
    }
}

//Address实体
@Entity
@Data
@AllArgsConstructor
@NoArgsConstructor
public class Address {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Integer id;
    private String street;
    private String city;

    /* mappedBy要和ApplicationUser实体类中的属性名一致也就是addressList, 
    这里使用懒加载, 不然会取每一个Address去找对应的User.*/
    @ManyToMany(mappedBy = "addressList", fetch = FetchType.LAZY)
    List<ApplicationUser> userList;

    @Override
    public String toString() {
        return "Address{" +
                "id=" + id +
                ", street='" + street + '\'' +
                ", city='" + city + '\'' +
                '}';
    }
}

多对多关联(使用复合主键)

使用复合主键完成多对多关联

  1. 创建复合主键类, 必须实现Serializable接口
@Embeddable //使用@Embeddable将UserAddressId定义为嵌入式组件
@Data
public class UserAddressId implements Serializable {

    @Column(name = "user_id")
    private Integer userId;
    @Column(name = "address_id")
    private Integer addressId;
}
  1. 创建关系表实体
@Entity
@Data
public class UserAddress {

    /*映射id主键, 将刚才定义的UserAddressId嵌入进来. 
    这里为复合主键, 因为在UserAddressId中定义了多个字段*/
    @EmbeddedId
    private UserAddressId id;

    @ManyToOne
    //映射id, 要和复合主键类中的属性名一致
    @MapsId("userId")
    //指定在关系表中的字段名称
    @JoinColumn(name = "user_id")
    private ApplicationUser applicationUser;

    @ManyToOne
    @MapsId("addressId")
    @JoinColumn(name = "address_id")
    private Address address;

    private Integer rating; //关系表中的字段
}

    @OneToMany(mappedBy = "applicationUser")

懒加载的情况下一共执行了3次查询, 因为在ApplicationUser中@OneToMany注解中没有指定FetchType,
点开@OneToMany的源码可以发现, 默认的FetchType为LAZY即为懒加载.

/** (Optional) Whether the association should be lazily loaded or
     * must be eagerly fetched. The EAGER strategy is a requirement on
     * the persistence provider runtime that the associated entities
     * must be eagerly fetched.  The LAZY strategy is a hint to the
     * persistence provider runtime.
     */
    FetchType fetch() default LAZY;

将ApplicationUser中@OneToMany指定为贪婪加载

@Data
@Accessors(chain = true)
@NoArgsConstructor
@AllArgsConstructor
@Entity
@TypeDef(name = "luckyNumbers", typeClass = JsonStringType.class)
public class ApplicationUser {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Integer id;

    private String name;
    private Integer age;
    private String email;

    @Type(type = "luckyNumbers")
    @Column(columnDefinition = "json")
    private int[] luckyNumbers;

    //指定fetchType为EAGER(贪婪加载)
    @OneToMany(mappedBy = "applicationUser", fetch = FetchType.EAGER)
    List<UserAddress> ratings;

    @Override
    public String toString() {
        return "ApplicationUser{" +
                "id=" + id +
                ", name='" + name + '\'' +
                ", age=" + age +
                ", email='" + email + '\'' +
                ", luckyNumbers=" + Arrays.toString(luckyNumbers) +
                '}';
    }
}
