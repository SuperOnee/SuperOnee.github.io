简介

概述

Gateway是在Spring生态系统之上构建的API网关服务, 基于Spring5, Spring Boot2和Project Reactor等技术. Gateway提供一种简单而有效的方式来对API进行路由, 以及提供一些强大的过滤器功能, 例如: 熔断, 限流, 重试等.

SpringCloud Gateway作为SpringCloud生态系统中的网关, 目标是代替zuul在SpringCloud2.0以上版本中, 没有对新版本zuul2.0以上最新高性能版本进行集成, 仍然还是用的zuul1.x非Reactor模式的老版本. 而为了提升网关的性能, SpringCloud Gateway是基于WebFlux框架实现的, 而WebFlux框架底层使用了高性能的Reactor通信框架Netty.

SpringCloud Gateway的目标提供统一的路由方式且基于Filter链的方式提供了网关基本的功能, 例如: 安全, 监控/指标, 和限流.

三大核心概念

  1. Route(路由)

    路由是构建网关的基本模块, 它由ID, 目标URL, 一系列的断言和过滤器组成, 如果断言为true则表示匹配该路由.

  2. Predicate(断言)

    参考的是java8的函数式中的Predicate接口开发人员可以匹配HTTP请求中的所有内容(例如请求头或者请求参数), 如果请求与断言相匹配则进行路由.

  3. Filter(过滤器)

    指的是Spring框架中gatewayFilter的实例, 使用过滤器可以在请求被路由前或者之后对请求进行修改.

web请求, 通过一些匹配条件, 定位到真正的服务节点, 并且在这个转发过程的前后, 进行一些精细化控制.

predicate就是我们的匹配条件, 而filter, 就可以理解为一个无所不能的拦截器. 有了这两个元素, 再加上目标uri, 就可以实现一个具体的路由了.

Predicate的使用

@Bean
public RouteLocator routes(RouteLocatorBuilder builder) {
    return builder.routes()
        //效果同配置文件相同
        .route("direct-route", r -> r.path("/provider/hello")	
               .and()
               //需要带上number这个请求参数, 才能访问
               .predicate(s -> s.getRequest().getQueryParams().get("number") != null)
               .uri("http://localhost:8080"))
        //路由到百度
        .route("direct-route2", r -> r.path("/").uri("https://www.baidu.com/"))
        .build();
}
  • 此时直接访问http://localhost:9527/provider/hello, 返回404.
  • 带上参数访问http://localhost:9527/provider/hello?number=1, 返回hello

yml方式配置Predicate

几种常用的Predicate:

  1. The After Route Predicate Factory

    spring:
  cloud:
    gateway:
      routes:
      - id: after_route
        uri: https://example.org
        predicates:
        - After=2017-01-20T17:42:47.789-07:00[America/Denver] #表示此接口在这时间之后才能访问, 中括号内表示时区[时区]

  2. The Before Route Predicate Factory

    spring:
  cloud:
    gateway:
      routes:
      - id: before_route
        uri: https://example.org
        predicates:
        - Before=2017-01-20T17:42:47.789-07:00[America/Denver] #表示此接口在这时间之前可以访问

  3. The Between Route Predicate Factory

    spring:
  cloud:
    gateway:
      routes:
      - id: between_route
        uri: https://example.org
        predicates:
        - Between=2017-01-20T17:42:47.789-07:00[America/Denver], 2017-01-21T17:42:47.789-07:00[America/Denver] #表示此接口在这两个时间区间可以访问

  4. The Cookie Route Predicate Factory

    spring:
  cloud:
    gateway:
      routes:
      - id: cookie_route
        uri: https://example.org
        predicates:
        - Cookie=chocolate, ch.p #表示需要携带cookie访问, 如果没有cookie则请求不到

  5. The Header Route Predicate Factory

    spring:
  cloud:
    gateway:
      routes:
      - id: header_route
        uri: https://example.org
        predicates:
        - Header=X-Request-Id, \d+ #表示请求头内需要有这些内容

  6. The Host Route Predicate Factory

    spring:
  cloud:
    gateway:
      routes:
      - id: host_route
        uri: https://example.org
        predicates:
        - Host=**.somehost.org,**.anotherhost.org #限制主机

  7. The Method Route Predicate Factory

    spring:
  cloud:
    gateway:
      routes:
      - id: method_route
        uri: https://example.org
        predicates:
        - Method=GET,POST #限制请求方法

  8. The Path Route Predicate Factory

    spring:
  cloud:
    gateway:
      routes:
      - id: path_route
        uri: https://example.org
        predicates:
        - Path=/red/{segment},/blue/{segment} #限制请求路径

  9. The Query Route Predicate Factory

    spring:
  cloud:
    gateway:
      routes:
      - id: query_route
        uri: https://example.org
        predicates:
        - Query=green #限制请求参数

  10. The RemoteAddr Route Predicate Factory

    spring:
  cloud:
    gateway:
      routes:
      - id: remoteaddr_route
        uri: https://example.org
        predicates:
        - RemoteAddr=192.168.1.1/24 #限制远程地址

  11. The Weight Route Predicate Factory

    spring:
  cloud:
    gateway:
      routes:
      - id: weight_high
        uri: https://weighthigh.org
        predicates:
        - Weight=group1, 8
      - id: weight_low
        uri: https://weightlow.org
        predicates:
        - Weight=group1, 2 #分组权重控制, 10个请求会有8个请求路由到weight_high, 两个路由到low.

全局Filter文档

根据官方示例手动实现一个GlobalFilter

  1. 在启动类中新增代码

    @SpringBootApplication
@EnableEurekaClient
@Configuration
@Slf4j
public class CloudGateWay9527Application {

    public static void main(String[] args) {
        SpringApplication.run(CloudGateWay9527Application.class, args);
    }

    @Bean
    public RouteLocator routes(RouteLocatorBuilder builder) {
        return builder.routes()
                //效果同配置文件相同
                .route("direct-route", r -> r.path("/provider/hello")
                        .and()
                        //需要带上number这个请求参数, 才能访问
                        .predicate(s -> s.getRequest().getQueryParams().get("number") != null)
                        .uri("http://localhost:8080"))
                //路由到百度
                .route("direct-route2", r -> r.path("/").uri("https://www.baidu.com/"))
                .build();
    }

    @Bean
    public GlobalFilter customFilter() {
        return new CustomGlobalFilter();
    }

    private static class CustomGlobalFilter implements GlobalFilter, Ordered {

        @Override
        public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
            log.info("custom global filter");
            String username = exchange.getRequest().getQueryParams().getFirst("username");
            if (!"nathan".equals(username)) {
                log.info("非法username: " + username);
                exchange.getResponse().setStatusCode(HttpStatus.NOT_ACCEPTABLE);
                return exchange.getResponse().setComplete();
            }

            return chain.filter(exchange);
        }

        @Override
        public int getOrder() {
            return -1;
        }
    }
}

  2. 启动项目测试

    • 请求参数没有增加username或username不等于nathan的情况下

    • 请求参数中增加username=nathan, 完整访问地址: http://localhost:9527/?username=nathan

    • 可以看到当参数正确后, 成功地路由到了百度.

  1. 修改service-provider-8080和service-provider-hystrix8081以及GateWay的配置
